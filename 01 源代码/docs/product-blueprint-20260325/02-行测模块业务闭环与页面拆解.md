# 行测模块业务闭环与页面拆解

## 1. 行测模块目标

行测模块需要完整支持三类训练模式：

- 每日打卡
- 自主刷题
- 模拟刷题

三类模式都应该共用“统一答题引擎”，但入口配置、提交写回、报告展示、记录归类需要区分。

## 2. 行测首页应该承载的内容

### 2.1 顶部固定区

- 考试类型选择
- 地区选择
- 学科切换
- 搜索入口
- AI 辅助入口

### 2.2 运营与快捷入口区

- Banner
- 每日一练
- 最新时政
- 历年试卷
- 套题练习
- 模拟考试

### 2.3 训练主区

- 考点树
- 上次练习继续入口
- 模拟考试模式区

## 3. 三大业务模式定义

### 3.1 每日打卡

定义：

- 系统每天固定生成一组题。
- 默认题量建议 5 题。
- 每天一组，按日期唯一。

核心特征：

- 入口是“日期列表”而不是“考点选择”。
- 提交后记录类型必须写为 `daily` 或 `checkin`。
- 用户可以看到哪些日期已完成、未完成、补打卡状态。

### 3.2 自主刷题

定义：

- 用户主动选择考点、数量、难度、年份等配置后开始训练。

核心特征：

- 入口是“参数配置 + 考点选择”。
- 提交后记录类型写为 `practice` 或 `self_practice`。
- 报告重点展示知识点维度、题型维度、正确率。

### 3.3 模拟刷题

定义：

- 用户以考试视角进行专项、混合或整卷模拟。

核心特征：

- 入口需要先选择模式，再做出题设置。
- 可选模式至少包括：专项、混合、套卷。
- 提交后记录类型写为 `mock` 或 `mock_exam`。
- 报告更偏考试结果和模块表现。

## 4. 统一答题引擎设计

建议不要做“三个完全不同的答题页”，而是做“一个统一答题引擎 + 一套上下文参数”。

### 4.1 统一上下文结构

```js
{
  subject: 'xingce',
  bizType: 'daily' | 'practice' | 'mock',
  sourceType: 'daily_task' | 'knowledge' | 'paper' | 'mixed',
  sourceId: 'xxx',
  title: 'xxx',
  knowledgeIds: [],
  paperId: '',
  questionCount: 20,
  duration: 20,
  difficulty: 'all',
  yearScope: 'all',
  questionIds: [],
  extraConfig: {}
}
```

### 4.2 统一答题页需要支持的能力

- 倒计时或计时
- 题号导航
- 答题卡
- 收藏
- 标记
- 交卷
- 暂存
- 断点续答
- 材料题展示
- 切换题目

### 4.3 统一答题页不应该负责的事情

- 决定题目来源
- 决定最终写到哪个记录分类
- 拼装复杂业务配置

这些应该由入口页或启动器处理好，再传入答题页。

## 5. 每日打卡完整闭环

### 5.1 页面链路

首页 -> 每日一练列表页 -> 当日入口 -> 统一答题页 -> 报告页 -> 解析页 -> 返回列表

### 5.2 页面要求

#### 页面一：每日一练列表页

职责：

- 按日期生成或拉取任务列表。
- 展示完成状态。
- 支持查看今天、历史日期、补练。

关键字段：

- 日期
- 星期
- 是否已完成
- 正确率
- 用时
- 是否可进入

#### 页面二：统一答题页

传入参数示例：

```js
{
  subject: 'xingce',
  bizType: 'daily',
  sourceType: 'daily_task',
  sourceId: 'daily-2026-03-25',
  title: '3月25日每日一练',
  questionCount: 5,
  duration: 10
}
```

#### 页面三：报告页

展示：

- 今日题量
- 正确数
- 用时
- 打卡状态
- 知识点命中情况
- 错题数量

#### 页面四：解析页

展示：

- 全部解析
- 仅错题解析

### 5.3 提交写回

每日打卡提交后至少要写入：

- 练习主记录
- 打卡状态
- 每日统计
- 错题集
- 用户累计数据

### 5.4 当前项目对应缺口

- 没有每日列表页。
- 没有按日期生成或获取任务的接口。
- 没有把提交结果归类为打卡数据。
- 没有打卡日历闭环。

## 6. 自主刷题完整闭环

### 6.1 页面链路

首页 -> 自主刷题配置弹层 / 页面 -> 考点树选择 -> 统一答题页 -> 报告页 -> 解析页 -> 练习记录

### 6.2 配置项建议

- 数量
- 难度
- 年份范围
- 考点
- 是否随机
- 是否只做错题

### 6.3 报告重点

- 本次考点
- 本次正确率
- 本次用时
- 错题列表
- 考点树命中

### 6.4 当前项目对应缺口

- 首页虽然有考点树，但点击后还未进入真实业务流。
- `practice` 页没有识别本次是哪个知识点、什么配置生成的。
- 提交后没有形成“自主刷题记录”。

## 7. 模拟刷题完整闭环

### 7.1 页面链路

首页 -> 模拟模式选择 -> 出题设置 -> 统一答题页或模考答题引擎 -> 报告页 -> 解析页 -> 练习记录

### 7.2 模拟模式定义

#### 专项

- 单一考点或单一模块

#### 混合

- 多个考点组合

#### 套卷

- 整套试卷

### 7.3 出题设置建议

- 时长
- 题量
- 是否随机顺序
- 是否按真实试卷顺序
- 试卷来源
- 是否展示模块分布

### 7.4 模考报告建议

必须展示：

- 总分或正确率
- 总用时
- 各模块正确率
- 各模块用时
- 各考点表现
- 错题解析入口
- 全部解析入口

### 7.5 当前项目对应缺口

- 首页的模考区块还没真正开始考试。
- `exam` 页的数据来源仍是页面内 Mock。
- 没有和首页的专项 / 混合 / 套卷选择真正打通。
- 没有完整报告沉淀到记录中心。

## 8. 行测记录中心设计

### 8.1 记录统一主表

建议所有行测练习都先写入一张练习主表：

```js
{
  recordId: 'rec_xxx',
  subject: 'xingce',
  bizType: 'daily' | 'practice' | 'mock',
  sourceType: 'daily_task' | 'knowledge' | 'paper' | 'mixed',
  sourceId: 'xxx',
  sourceName: 'xxx',
  questionCount: 20,
  correctCount: 15,
  accuracy: 75,
  durationSeconds: 1120,
  submitTime: '2026-03-25 21:30:00',
  knowledgeIds: [],
  paperId: '',
  reportId: 'report_xxx'
}
```

### 8.2 我的练习记录页

要求：

- 按日期倒序
- 支持按类型筛选：打卡 / 自主刷题 / 模拟刷题
- 支持按时间筛选
- 支持按知识点 / 试卷名称检索
- 点击进入报告

### 8.3 我的错题页

要求：

- 支持按知识点筛选
- 支持按试卷筛选
- 支持重练
- 支持查看解析

### 8.4 我的收藏页

要求：

- 支持题目收藏
- 支持试卷收藏
- 支持继续练习

### 8.5 我的报告页

要求：

- 累计做题数
- 累计正确率
- 连续学习天数
- 每日做题趋势
- 考点树统计

## 9. 行测模块实施优先级

建议顺序如下：

1. 先做统一答题上下文和统一记录主表。
2. 再做每日打卡闭环。
3. 再做自主刷题闭环。
4. 再做模拟刷题闭环。
5. 最后补我的记录、错题、收藏、统计。

## 10. 行测模块验收标准

当以下条件全部成立时，才能认为行测模块完成了一期闭环：

1. 首页可以分别进入每日打卡、自主刷题、模拟刷题。
2. 三种入口都能进入统一答题页。
3. 交卷后都能生成对应报告。
4. 报告能进入解析。
5. 记录页能按类型看到三种结果。
6. 错题页能看到本次新增错题。
7. 我的首页统计会跟着变。
