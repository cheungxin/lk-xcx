# 领考小考 (LK-XCX) 后端接口开发文档

## 1. 文档声明
本文件基于当前项目 Mock 数据及前端业务逻辑编写，主要涵盖 **首页、用户、行测** 等核心模块。旨在为后端提供标准化接口定义。

### 1.1 基础信息
- **Base URL**: `https://api.lingkao.com/v1`
- **数据格式**: `application/json`
- **公共状态码**: `200(成功)`, `401(未授权)`, `403(无权)`, `500(服务器错误)`

---

## 2. 首页模块 (Home Module)

### 2.1 获取首页全局配置
- **接口**: `GET /home/config`
- **描述**: 初始化首页所需考试类型、可选地区及固定入口。
- **响应**: `examTypes`, `subjectTabs`, `quickEntries` (见 Mock 结构)。

### 2.2 获取轮播图 (Banners)
- **接口**: `GET /home/banners`
- **参数**: `examType` (可选)
- **字段**: `id`, `image`, `url`, `type`(page/web), `title`。

---

## 3. 用户模块 (User Module)

### 3.1 获取用户信息
- **接口**: `GET /user/info`
- **字段**: `userId`, `nickname`, `avatar`, `phone`, `studyDays`, `accuracy`, `rank` 等。

### 3.2 学习统计 (日报/周报)
- **接口**: `GET /user/statistics`
- **内容**: 今日/本周/本月/累计的 `questions`, `correct`, `accuracy`。

### 3.3 学习进度与勋章
- **接口**: `GET /user/progress` 及 `GET /user/badges`。

### 3.4 提交用户反馈
- **接口**: `POST /user/feedback`
- **字段**: `content` (正文), `contact` (联系方式), `images` (图片列表)。

---

## 4. 行测模块 (Xingce Module)

### 4.1 题库分类与题量 (考点)
- **接口**: `GET /practice/categories`
- **参数**: `examType`, `region`
- **特性**: **总题数 (`total`) 应根据所选考试类型和地区实时动态计算。**

### 4.2 题目列表 (专项练习)
- **接口**: `GET /practice/questions`
- **参数**: `categoryId`, `limit`, `examType`, `region`。
- **题目扩展字段**: 
  - `score`: 分数值
  - `difficulty`: 难度 (`easy`/`medium`/`hard`)
  - `source`: 来源 (如 "2024国考")
  - `year`: 年份 (如 2024)
  - `material`: 共享材料 (文字/图片)

### 4.3 真题/模拟卷列表
- **接口**: `GET /papers/list`
- **参数**: `type`(recommend/historical/full), `examType`, `region`。

### 4.4 试卷详情
- **接口**: `GET /papers/{id}`
- **响应**: 包含试卷基本信息及各板块 (`sections`) 下的题目。

### 4.5 互动功能 (收藏与标记)
- **收藏接口**: `POST /practice/collect` (参数: `questionId`, `isCollect`)
- **获取收藏**: `GET /user/collections`
- **答题标记**: `POST /practice/mark` (参数: `recordId`, `questionId`, `isMark`) -> 用于标记考试中不确定的题目。

### 4.6 答题记录提交与刷题报告
- **提交接口**: `POST /practice/submit`
  - 请求包含耗时、选项列表。
  - **响应**: 返回 `recordId`。
- **报告接口**: `GET /practice/report/{recordId}`
  - 核心字段：`totalScore`, `accuracy`, `rank`(排名), `beatRatio`(击败比例), `sectionsStat`(各版块得分情况)。

---

## 5. 开发建议与扩展性

1.  **索引优化**: 对 `Question` 表的 `examType`, `region`, `categoryId`, `year` 建立复合索引以保障筛选性能。
2.  **一材多题关联**: 后端需在 `Question` 表中维护 `material_id`，若多题 ID 相同，前端呈现为材料题组。
3.  **计算延迟**: 刷题报告中的“击败百分比”等全站数据，建议采用定时任务或从 Redis 缓存统计表中读取，而非实时扫描全表。
4.  **接口兼容**: 答案 `correctAnswer` 应同时兼容字符串（单选）和数组（多选）。
