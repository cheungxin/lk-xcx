# 申论模块业务闭环与页面拆解

## 1. 申论模块定位

申论不能简单复用行测首页内容，只能复用“产品骨架”和“练习记录体系”。

申论和行测的差异决定了它必须有独立的业务首页和独立的作答渲染方式：

- 行测是客观题主导，结果即时判定。
- 申论是材料题和文本作答主导，作答过程更长，数据结构更复杂。

因此建议申论采用：

- 统一外壳
- 独立首页
- 独立作答渲染器
- 共用记录中心

## 2. 申论首页建议结构

根据你的口径，申论首页下方主入口建议改为：

- 热点素材
- 规范表述
- 随机出题
- 自选题目

可选扩展：

- 历史作答
- 精选范文
- 申论批改记录

## 3. 四大核心入口

### 3.1 热点素材

定位：

- 用户日常积累素材、政策热点、案例论据。

页面链路：

申论首页 -> 热点素材列表 -> 素材详情 -> 收藏 / 关联练习

最低能力要求：

- 列表
- 分类
- 搜索
- 收藏
- 最近浏览

### 3.2 规范表述

定位：

- 用户积累高频规范表达、标题写法、开头结尾模板、对策句式。

页面链路：

申论首页 -> 规范表述列表 -> 详情页 -> 收藏 / 复制 / 关联题目

最低能力要求：

- 按主题分类
- 标签筛选
- 收藏
- 复制

### 3.3 随机出题

定位：

- 类似行测自主刷题，但题目载体是申论题型和材料。

可配置项：

- 考点
- 题型
- 难度
- 数量

页面链路：

申论首页 -> 随机出题配置 -> 申论作答页 -> 作答报告 -> 解析 / 参考答案

### 3.4 自选题目

定位：

- 用户明确选择一组材料和题目后作答。

页面链路：

申论首页 -> 题目列表 / 材料列表 -> 题目详情 -> 申论作答页 -> 作答报告

最低能力要求：

- 按主题、题型、年份筛选
- 查看材料摘要
- 查看题目要求
- 开始作答

## 4. 申论作答页设计建议

申论作答页不建议直接套用行测的选项式交互。

建议采用“通用答题外壳 + 申论内容区域”的方式。

### 4.1 外壳共用能力

- 计时
- 暂存
- 交卷
- 材料目录
- 题号切换
- 返回确认

### 4.2 申论内容区域独有能力

- 材料阅读
- 题目要求展示
- 文本输入
- 字数统计
- 草稿区
- 参考要点查看权限控制
- 可能的 AI 评价入口

### 4.3 申论答题上下文

```js
{
  subject: 'shenlun',
  bizType: 'random' | 'custom' | 'mock',
  sourceType: 'topic' | 'paper' | 'material_group',
  sourceId: 'xxx',
  title: '乡村振兴专题申论练习',
  materialIds: ['m1', 'm2'],
  questionIds: ['q1', 'q2'],
  duration: 90,
  config: {
    showWordCount: true,
    allowDraft: true
  }
}
```

## 5. 申论报告页设计建议

由于申论未必有完全自动判分，报告页建议分两个层级。

### 5.1 即时报告

- 作答题数
- 用时
- 字数
- 是否已提交
- 参考答案入口

### 5.2 扩展报告

如果后续引入 AI 评价或人工批改，可继续增加：

- 结构完整度
- 观点覆盖度
- 语言规范度
- 分题评价

## 6. 申论记录、收藏、错题设计

### 6.1 记录

申论记录应该和行测共用主记录结构，但字段扩展不同。

增加字段建议：

- materialTitle
- questionType
- wordCount
- draftCount
- evaluationStatus

### 6.2 收藏

申论收藏不只是题目收藏，还可能包括：

- 素材收藏
- 规范表述收藏
- 题目收藏

### 6.3 错题

申论的“错题”概念建议谨慎定义。

更合理的方式是：

- 待巩固题目
- 低质量作答题目
- 需要重写题目

如果后面引入评分体系，再映射到“错题”概念。

## 7. 当前项目对应缺口

目前申论的缺口是整套业务体系，而不是单个页面：

- 没有申论首页。
- 没有热点素材页。
- 没有规范表述页。
- 没有随机出题配置页。
- 没有自选题列表页。
- 没有申论作答页。
- 没有申论报告页。
- 没有申论记录与收藏结构。

## 8. 申论模块推荐 MVP 范围

为了避免申论一期做得太重，建议 MVP 只做四件事：

1. 申论首页
2. 随机出题
3. 自选题目
4. 作答记录

热点素材和规范表述可以做成二期补充，因为它们更像内容库能力。

## 9. 申论模块实施顺序

建议顺序如下：

1. 先设计申论首页与数据模型。
2. 再做题目列表 / 材料列表。
3. 再做申论作答页。
4. 再做即时报告。
5. 最后做素材库、规范表述、收藏、扩展评价。

## 10. 申论模块验收标准

申论一期至少需要满足：

1. 首页可以区分于行测首页。
2. 用户可以进入随机出题和自选题两个入口。
3. 用户可以阅读材料并完成作答。
4. 提交后可以形成记录。
5. 我的页面可以看到申论维度的记录。
