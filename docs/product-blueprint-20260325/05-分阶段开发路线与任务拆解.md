# 分阶段开发路线与任务拆解

## 1. 目标

这份路线图用于把“大而全的需求”拆成可以执行的开发批次。建议严格按阶段推进，不建议跳阶段开发。

## 2. 总体实施策略

### 2.1 先骨架，后页面

优先完成：

- 路由与页面注册统一
- 答题上下文统一
- 记录主表统一
- 接口分组统一

再去补页面细节。

### 2.2 先行测，后申论

原因：

- 行测更适合先打通闭环。
- 申论依赖更复杂的数据结构。
- 我的模块需要先有前面练习闭环的真实数据来源。

### 2.3 每个阶段都要有验收结果

每一阶段都应该输出：

- 完整页面链路
- 数据写回结果
- 验收清单

## 3. 阶段拆解

## 阶段 0：架构整理

目标：

- 把当前原型工程整理成可持续开发的基线。

任务：

1. 修正 `app.json` 页面注册与路由常量不一致的问题。
2. 明确底部导航最终结构。
3. 统一答题上下文字段。
4. 统一记录主表字段。
5. 按业务重构 `interface/api.js`。

交付物：

- 统一路由表
- 统一答题上下文定义
- 统一业务接口文件

验收标准：

- 所有正式页面都已注册。
- 不再出现“页面存在但未注册”的情况。
- 页面可以通过统一参数进入。

## 阶段 1：行测每日打卡闭环

目标：

- 先打通最短、最标准的一条刷题闭环。

任务：

1. 新增每日一练列表页。
2. 接入每日任务接口或本地模拟接口。
3. 从日期列表进入统一答题页。
4. 提交后生成报告页。
5. 报告可进入解析页。
6. 写入练习记录与打卡状态。

交付物：

- 每日一练列表页
- 每日打卡启动逻辑
- 每日报告页

验收标准：

- 用户可以完成今天的 5 题。
- 完成后能在记录中看到“每日打卡”。
- 日历或列表状态变为已完成。

## 阶段 2：行测自主刷题闭环

目标：

- 把考点树和练习配置真正接入业务。

任务：

1. 首页考点树点击进入自主刷题配置。
2. 支持数量、难度、年份、考点配置。
3. 启动统一答题页。
4. 提交后写入自主刷题记录。
5. 报告页展示考点维度结果。

交付物：

- 自主刷题启动页或弹层
- 自主刷题报告页

验收标准：

- 用户能按考点定制练习。
- 提交后记录页可筛选到“自主刷题”。

## 阶段 3：行测模拟刷题闭环

目标：

- 完成专项 / 混合 / 套卷三种模考模式。

任务：

1. 首页模考模式选择可用。
2. 模考出题设置可用。
3. 模考启动后进入统一答题引擎或模考答题壳。
4. 交卷后生成完整报告。
5. 记录中可筛选“模拟刷题”。

交付物：

- 模考配置页
- 模考报告页
- 模考记录落地

验收标准：

- 三种模式都能启动。
- 报告包含模块统计、用时、解析入口。

## 阶段 4：我的模块数据化

目标：

- 把“我的”从页面原型变成数据中心。

任务：

1. 我的首页接真实统计接口。
2. 练习记录接真实列表接口。
3. 错题本接真实筛选逻辑。
4. 收藏页接真实数据。
5. 报告页接累计统计。
6. 增加行测 / 申论切换能力。

交付物：

- 数据化我的首页
- 记录页
- 错题页
- 收藏页
- 统计页

验收标准：

- 提交练习后，我的页面数据会实时变化。
- 记录、错题、收藏之间跳转顺畅。

## 阶段 5：申论一期闭环

目标：

- 建立申论最小可用业务链路。

任务：

1. 新建申论首页。
2. 新建随机出题入口。
3. 新建自选题目入口。
4. 新建申论作答页。
5. 新建申论即时报告。
6. 申论记录进入我的。

交付物：

- 申论首页
- 申论作答页
- 申论记录页能力

验收标准：

- 用户能完成一次申论随机题或自选题作答。
- 我的里可以看到申论记录。

## 阶段 6：申论内容库与高级能力

目标：

- 补全申论内容生态。

任务：

1. 热点素材库。
2. 规范表述库。
3. 素材收藏。
4. 表述收藏。
5. 扩展评价能力。

## 4. 建议的任务分配粒度

如果后续你要把任务派给其他 AI，建议一个任务只做一个闭环，不要混做。

推荐拆法：

- 任务包 A：统一答题上下文和路由修正
- 任务包 B：每日一练列表 + 启动
- 任务包 C：统一报告页
- 任务包 D：自主刷题配置页
- 任务包 E：模考配置页
- 任务包 F：记录页数据化
- 任务包 G：错题页数据化
- 任务包 H：申论首页
- 任务包 I：申论作答页

## 5. 每个任务包都要附带的输入

你后续给 AI 发任务时，建议每次都附上这 4 类信息：

1. 本次要做的页面或模块。
2. 本次要遵守的数据字段。
3. 本次的页面跳转起点和终点。
4. 本次的验收标准。

## 6. 任务验收模板

每个任务完成后，建议按下面模板验收：

```md
### 任务名称
- 范围：
- 入口：
- 出口：
- 写入数据：
- 验收点 1：
- 验收点 2：
- 风险：
```

## 7. 当前最优先的前三个任务

如果只选三个任务立刻开始，我建议是：

1. 阶段 0：统一答题上下文、页面注册、接口分组
2. 阶段 1：行测每日打卡闭环
3. 阶段 2：行测自主刷题闭环

因为这三个任务完成后，你的小程序就不再只是原型，而是开始具备真正的业务骨架。
