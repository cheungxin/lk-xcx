# 实现计划：草稿画板组件 Canvas 优化

## 概述

本实现计划将草稿画板组件从旧版 Canvas API 升级到 Canvas 2D API，修复坐标转换问题，支持高 DPI 屏幕，优化绘画性能。实现将分为六个主要阶段：Canvas 2D API 升级、坐标转换实现、绘画引擎实现、错误处理、性能优化和测试。

## 任务

- [x] 1. 升级 Canvas 2D API 和初始化逻辑
  - [x] 1.1 修改 WXML 模板，将 Canvas 从旧版升级到 Canvas 2D
    - 将 `<canvas canvas-id="draft-canvas">` 改为 `<canvas type="2d" id="draft-canvas">`
    - 保持 CSS 类名和样式不变
    - _需求: 1.1, 1.4_

  - [x] 1.2 实现 Canvas 2D 初始化函数 `initializeCanvas2D()`
    - 使用 `wx.createSelectorQuery()` 查询 Canvas 节点和边界矩形
    - 获取 Canvas 2D 上下文 `canvas.getContext('2d')`
    - 实现重试机制（最多 5 次，间隔 50ms）
    - 返回包含 ctx、canvas、dpr、rect 的结果对象
    - _需求: 1.2, 1.3, 2.1, 2.2, 2.3, 2.4, 2.5_

  - [x] 1.3 实现高 DPI 屏幕适配逻辑
    - 获取设备像素比 `wx.getSystemInfoSync().pixelRatio`
    - 设置 Canvas 物理尺寸为逻辑尺寸乘以 DPR
    - 调用 `ctx.scale(dpr, dpr)` 缩放上下文
    - _需求: 3.1, 3.2, 3.3, 3.4, 3.5_

  - [x] 1.4 编写 Canvas 初始化的属性测试
    - **属性 1: Canvas 初始化正确性**
    - **验证需求: 2.1, 2.4, 3.1, 3.2, 3.3**
